# BIMA Operating System — Year 1

Dokumen ini adalah sistem operasi kerja BIMA untuk periode April 2026 sampai Maret 2027.
Tujuannya sederhana: memastikan tiga engine BenihSaham bergerak paralel tanpa melanggar mandat board.

## North Star

- Bangun mesin kepercayaan: portofolio riil, dokumentasi jujur, dan edukasi yang konsisten.
- Kejar target revenue Rp100 juta tanpa mengorbankan integritas edukasi.
- Jadikan setiap keputusan investasi sebagai aset pembelajaran dan bahan konten.

## Guardrails yang Tidak Bisa Dilanggar

1. Tidak memberikan rekomendasi saham spesifik dalam produk berbayar.
2. Tidak membuat klaim return tanpa data aktual.
3. Tidak mengalokasikan modal di atas Rp2 juta tanpa persetujuan board.
4. Tidak mengubah strategi investasi inti tanpa keputusan board.
5. Tidak mempublikasikan Annual Letter tanpa persetujuan board.

## Operating Rhythm

### Daily — 08.00 WIB

- Cek IHSG, posisi portofolio, dan berita pasar semalam.
- Catat aksi yang diperlukan di [portfolio/daily/daily-market-check-template.md](/Users/fanioz/code/BenihSaham/portfolio/daily/daily-market-check-template.md).
- Jika tidak ada aksi portofolio, alihkan fokus ke produksi konten atau revenue sprint hari itu.

### Weekly — Senin

- Review performa konten minggu lalu.
- Tetapkan dua konten minggu berjalan: satu edukasi evergreen dan satu proses/portfolio log.
- Perbarui watchlist dan peluang valuasi mingguan.
- Perbarui [reports/board/CEO_TASK_BOARD.md](/Users/fanioz/code/BenihSaham/reports/board/CEO_TASK_BOARD.md) agar board bisa melihat prioritas aktif, blocker, dan tugas yang selesai.

### Monthly — Akhir bulan

- Review semua pegangan, thesis, dan posisi kas.
- Perbarui trading plan bulan depan dan Plan B.
- Review KPI content dan revenue.
- Kirim laporan board dengan format Portfolio / Content / Revenue / Flagged.

### Quarterly — Apr / Jul / Okt / Mar

- Screening laporan keuangan terbaru untuk universe saham yang dipantau.
- Refresh thesis watchlist dan revisi investment plan.
- Terbitkan laporan kuartalan publik untuk audiens.

### Annual — Januari

- Susun Annual Letter berbasis data aktual.
- Ajukan ke board untuk approval sebelum publikasi.

## Tiga Engine

### Portfolio Engine

- Input utama: laporan keuangan, data pasar, dan catatan manajemen.
- Workflow keputusan: screen -> valuasi -> cek fundamental -> cek manajemen -> putuskan buy/hold/wait/reject -> catat reasoning.
- Output utama: jurnal keputusan, watchlist, review bulanan, screening kuartalan.

### Content Engine

- Input utama: keputusan portofolio, pertanyaan audiens, dan konsep value investing.
- Distribusi utama: YouTube, website, Substack.
- Ritme minimum: dua konten per minggu.
- Pilar: metodologi, studi kasus, jurnal portofolio, kesalahan, risiko dan ekspektasi realistis.

### Revenue Engine

- Urutan monetisasi: bangun trust -> validasi minat -> jual produk edukasi -> stabilkan membership -> tambah tools sederhana.
- Funnel default: konten gratis -> newsletter -> entry product -> membership.
- Fokus tahun pertama: monetisasi metodologi dan proses, bukan sinyal beli-jual.

## Delegasi Sub-agent

- Research Agent: screening emiten, rangkum LK, update watchlist.
- Content Writer Agent: artikel web, newsletter, dan outline edukasi.
- Video Script Agent: skrip YouTube berbasis jurnal portofolio.
- SEO Agent: optimasi artikel pilar dan topical authority.
- Data Agent: dashboard KPI, spreadsheet valuasi, dan rekap laporan.

## Definition of Done

- Mingguan: dua konten terjadwal atau terbit, watchlist ter-update, dan heartbeat harian tidak miss.
- Bulanan: board report terkirim, semua pegangan punya thesis aktif, KPI content dan revenue tercatat.
- Kuartalan: screening selesai, evaluasi portfolio vs thesis selesai, dan prioritas bisnis kuartal berikutnya diputuskan.
- Tahunan: semua KPI dapat diukur dengan data nyata dan Annual Letter siap diajukan ke board.

## Visibility to Board

- Board memonitor kerja harian CEO melalui [reports/board/CEO_TASK_BOARD.md](/Users/fanioz/code/BenihSaham/reports/board/CEO_TASK_BOARD.md).
- Laporan bulanan tetap menjadi ringkasan resmi, tetapi task board menunjukkan pekerjaan yang sedang berlangsung di antara dua laporan.
